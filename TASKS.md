## 完了済みタスク

### v0.8 - UI/UX極大改善 & メッセージ管理・レイアウト刷新 (2026-05-26)
- [x] **メッセージ管理機能の完全実装 (編集/削除/再生成)**
  - メッセージホバー時にアクションボタン（✏️編集, 🗑️削除, 🔄再生成）を表示。
  - 長文対応のメッセージ編集用モーダルダイアログを実装。
  - APIエラー時にユーザーの入力を消失させない（popしない）安全ガード機能。
  - エラー発生時、またはユーザー入力直後に「AIの応答を生成 (リトライ)」ボタンを表示する機能。
- [x] **チャットレイアウトと視認性の劇的改善**
  - 地の文（ナレーション）のDOM構造を会話吹き出しと統一し、非表示アバター枠（スペーサー）を配置することで、左端の開始ラインをセリフとピクセル単位で完全同期。
  - PC・タブレット等の大画面表示時、タイムライン全体を中央寄せのカラム（max-width: 800px）に制限し、視線移動の疲労を軽減。
  - ナレーションの背景色・文字色・透明度をユーザーが設定画面から自由にカスタマイズできる機能。
- [x] **操作性とテキスト入力の最適化**
  - 世界観設定やプロンプト指示、ユーザー入力欄（textarea）を入力文字量に応じて自動拡張（Auto-resize）する機能。
  - 自動スクロールのON/OFF切り替え設定を追加。
  - フォントサイズ設定をプルダウンから「任意の数値入力 (px)」に変更し、CSS変数(`:root`)を用いてページ全体へリアルタイムに適用。
  - 送信キーバインドの最適化（スマホのEnterは改行、PCのCtrl+Enterで送信）。
- [x] **大画面トリミング（クロッパー）機能の高度化**
  - アバター設定時、アイコンの極小枠内ではなく、画面中央に大きな独立トリミングモーダルを表示。
  - プレビュー画像のアスペクト比を動的に計算し、操作時のズレを完全に解消。
  - 一度アップロードした画像をメモリに保持し、「位置を再調整」ボタンから再アップロードなしで何度でもトリミングをやり直せる機能。
- [x] **Dropbox同期とState管理の堅牢化**
  - `isSyncInProgress` による排他制御（Mutex）を導入し、起動時のDropbox自己ロック（「他の端末が同期中です」エラー）を完全排除。
  - タブ復帰時（Visibility API）の同期に「10分間のスロットル（間引き）制御」を導入し、無駄なAPI通信を削減。
  - 最後に開いていたストーリー（`last_active_story_id`）を記憶し、アプリ再起動時に自動復元する機能。
