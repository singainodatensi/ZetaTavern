# ZetaTavern - キャラクターシステム仕様 (Character System)

最終更新: 2026-05-22

---

## 概要

ZetaTavernのキャラクターシステムは、ユーザー登録キャラとAI生成キャラを統合的に扱うハイブリッド方式を目指す。

---

## 現在の実装

### キャラクターデータモデル
```javascript
{
  characterId: string,       // UUID
  name: string,              // キャラクター名
  description: string,       // 詳細設定・背景
  personality: string,       // 性格キーワード
  mes_example: string,       // 台詞サンプル
  avatarAssetId: string      // アバター画像のアセットID
}
```

### 登場役割 (Attendance)
| 役割 | プロンプトへの載せ方 |
|------|---------------------|
| 主要 (main) | 設定・口調サンプルまで全文 |
| 補助 (support) | 要約 + 性格 |
| 背景 (background) | 名前と「その場にいる」程度 |
| 不在 (absent) | プロンプトから除外 |

### チャット表示でのキャラクター表示
1. AI出力パーサーがキャラクター名を検出
2. キャラクターライブラリとファジーマッチ
   - 完全一致 → そのキャラのアバター表示
   - 部分一致（名前の一部が含まれる）→ マッチ
3. 未登録キャラ → **デフォルトシルエット** (`assets/default-silhouette.png`) で表示

---

## 方針：ハイブリッド方式

### 方式A: デフォルト表示（実装済み）
- ライブラリに存在しないキャラ名 → デフォルトシルエットアイコン
- 一般NPC / モブとして扱う
- Zeta AI に近い方式

### 方式B: AI自動追加（将来実装）
- AIが登場させた未登録キャラを自動的にライブラリに追加
- `autoGenerated: true` フラグで区別
- ユーザーが後からアバター・設定を肉付け可能

### 推奨フロー（将来の完成形）
```
1. チャットパーサーで登場名を検出
2. ライブラリに一致 → そのアバター使用
3. 未登録 → シルエットで表示
4. （自動追加モード有効時）→ 暫定キャラとしてリストに追加
   - attendance: 'background'
   - autoGenerated: true
5. ユーザーが編集画面で正式登録
```

## キャラクター名のマッチング

### 現在の実装
```javascript
// 完全一致
characters.find(c => c.name === speakerName);

// 部分一致（どちらかが相手を含む）
characters.find(c => 
  c.name.includes(speakerName) || speakerName.includes(c.name)
);
```

### 将来の強化案
- `aliases[]` フィールドの追加（別名・ニックネーム）
  - 例: `{ name: "中野四葉", aliases: ["四葉", "よつば"] }`
- ひらがな/カタカナの正規化マッチ
- 姓/名の分離マッチ

## カテゴリ問題（検討中）

### 現在のカテゴリ
主要 / 補助 / 背景 / 不在

### 懸念点
- ユーザー側の管理負荷
- AI側の更新負荷
- 「補助」と「背景」の差がAIに伝わりにくい

### 検討中の改善案
- 短期: UIラベルを簡略化（「登場する / しない」+ 詳細度）
- 中期: 「登場」フラグ + 詳細度（高/低）の2軸に整理
- プロンプト構築ロジックは役割名が変わっても同じ考え方でOK

---

## AI エージェント向け注意事項

キャラクターシステムを編集・拡張する際:
- `js/ui.js` の `parseModelOutputToSegments()` がチャットパーサー
- `js/ui.js` の `matchCharacterByName()` がマッチングロジック
- `js/ai-client.js` の `buildSystemInstruction()` がプロンプト構築
- `js/db.js` の `characters` ストアがデータ永続化
- アバター画像は `assets` ストアに Blob として保存
- デフォルト画像は `assets/default-silhouette.png`
